# MonitorFlow — 智能监控告警流水线

## 04 字段文本

我构建了一个基于多 Agent 长链推理的智能监控告警 Agent——**MonitorFlow**。该项目解决的核心痛点是：监控系统产生海量告警噪音，真正的问题被淹没在无关告警中，告警阈值设置依赖人工经验缺乏自适应能力，且告警到根因定位的链路过长导致应急响应慢。核心逻辑流采用四层 Agent 长链推理架构，第一层 Metric Collector Agent 自动采集多源监控数据（Prometheus/Grafana/CloudWatch/ELK），构建统一的时间序列数据湖；第二层 Anomaly Detector Agent 基于统计学习和时序分析自动识别异常模式，动态调整告警阈值减少误报，输出结构化异常画像；第三层 Correlation Engine Agent 对多维度告警进行关联分析，识别告警风暴和级联故障模式，聚合相关告警降低告警数量；第四层 Root Cause Advisor Agent 综合前三层分析定位根因，生成告警处置建议和应急预案，支持自动触发 Runbook 执行修复操作。系统采用 DeepSeek API 驱动长链推理，Agent 间全部通过结构化 JSON 通信形成可追溯、可审计的监控链路。目前已在个人监控平台中落地使用，单次完整流水线运行消耗约 600 万 Token，将告警噪音降低 80%，告警响应时间缩短 70%，根因定位准确率达到 85% 以上。

## 版本三
